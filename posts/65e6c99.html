一、Vuejs简介

  1. 认识vuejs

    • 为什么学习vuejs
      • 对现有项目使用vue进行重构
      • 公司新项目决定使用vue技术栈
      • 招聘要求
    • 简单认识vuejs
      • Vue(读音/vju:/) 不要读错
      • Vue是一个渐进式框架
        • 渐进式意味着可以将vue作为应用的一部分嵌入其中，带来更丰富的交互体验
        • 如果希望更多业务逻辑使用vue实现，vue和辛苦及其生态系统可以满足各种各样的需求
          • 如Core+Vue-router+Vues
      • Vue有很多特点和web开发中常见的高级功能
        • 解耦视图和数据
        • 可复用组件
        • 前端路由技术
        • 状态管理
        • 虚拟DOM
      • 学习vuejs前提
        • HTML CSS JavaScript

  2. vuejs安装方法

    • CDN引入

      • <!-- 开发环境版本，包含了有帮助的命令行警告 -->
<script src="https://cdn.jsdelivr.net/npm/vue/dist/vue.js"></script>

    • 下载和引入

      • 开发环境 https://vuejs.org/js/vue.js
生产环境 https://vuejs.org/js/vue.min.js

    • 创建Vue对象时候传入了一些options:{}

      • {}中包含了el属性：该属性决定了这个vue对象挂载到哪一个元素上，很明显，挂id为app的div上
      • {}包含data属性：属性中存储一些数据
        • 这些数据可以由我们直接定义
        • 也可以来自网络从服务器加载

    • 列表

    • 计数器

  3. vuejs中的MVVM

    • MVVM

      Model View ViewModel，dom页面和数据通过vue进行通信

    • View层

      视图层，在我们前端开发中，通常就是DOM层，给用户展示各种信息

    • Model层

      数据层，数据可能是我们固定死的数据，更多的是来自我们服务器，从网络上请求下来的数据

    • VueModel层

      视图模型层，视图模型层是View和Model沟通的桥梁，一方面它实现了Data Binding,数据绑定，将Model的改变实时反应到View中，另一方面它实现了DOM Listener，也就是DOM监听，当DOM发生一些事件时，改变对应的Data

  4. 创建Vue实例传入的options

    • el：
      • 类型：String |HTMLElement
      • 作用：决定之后Vue实例会管理哪一个DOM
    • data：
      • 类型：Object|Function
      • 作用：Vue实例对应的数据对象
    • methods：
      • 类型：{ { [key:string]:Function} }
      • 作用：定义属于Vue的一些方法，可以在其他地方调用，也可以在指令中使用

四、事件监听

v-on介绍

  • 作用：绑定事件监听器
  • 缩写：@
  • 预期：function|Inline Statement| Object
  • 参数：event

v-on基本使用

v-on参数

如果不需要传参，可以省略()

如果省略()，方法本身中有一个参数，默认将原生event参数传递

如果需要同时传入某个参数，还需要event，可以通过$event传入事件

v-on修饰符

  • .stop - 调用 event.stopPropagation()
  • .prevent - 调用 event.preventDefault()
  • .{keyCode|keyAlias} - 只当事件是从特定键触发时才触发回调
  • .native - 监听组件根元素的原生事件
  • .once - 只触发一次回调

v-if、v-else-if、v-else

vue的条件指令可以根据表达式的值在DOM中渲染或销毁元素或组件

v-if原理：

  • v-if后面的条件为false时，对应的元素以及其子元素 不会渲染
  • 也就是不会有对应标签出现在DOM中

基本不用，计算属性不香吗。

登陆切换小案例

案例问题：

  • 如果我们在有输入内容的情况下，切换类型，文字依然显示之前输入的内容
  • 但是按道理讲，我们应该切换到另一个input元素中了
  • 在另一个input元素我们并没有输入内容
解答：
  • Vue在进行DOM渲染时，出于性能考虑，会尽可能复用已经存在的元素，而不是创建新元素
  • 在上面案例，Vue内部会发现原来的input元素不再使用，直接作为else中的input来使用了
解决：
  • 如果我们不希望vue出现类似重复利用问题，可以给对应input添加key，并保证key不同

v-show

  • v-show用法与v-if相似，也用于决定一个元素是否渲染
  • v-if和v-show对比
  • v-i和v-show都可以决定一个元素是否渲染，那么开发中我们如何选择呢？
    • v-if当条件为false时，压根不会有对应元素在DOM中
    • v-show当条件为false时，将元素display设成none
  • 选择
    • 需要显示隐藏频繁切换 v-show
    • 只有一次切换，v-if

v-for遍历

组件key属性

  • 官方推荐使用v-for时，给对应元素或组件添加:key属性
  • 作用主要为了高效更新虚拟DOM

响应式数组方法

push() ：数组末尾添加元素

pop() ：删除数组末尾元素

shift() ：删除数组第一个元素

unshift()：在数组最前面添加元素

splice() ：

  • 删除：第一个参数是开始位置，第二个是从开始位置起删除几个
    • 不传第二个表示从开始位置起删除后面所有元素（保留前几个）
  • 插入：第一个参数开始位置，第二个删除个数为0，第三个要添加的元素
  • 替换：第一个参数开始位置，第二个删除的个数，第三个要添加的元素

sort() ：排序

reverse() ：内容反转

Vue.set() ：第一个参数修改对象，第二个索引值（修改的元素下标），第三个修改内容

数组高阶函数

表单绑定v-model

  • v-model实现表单元素和数据双向绑定

  • 解析：当我们在输入框输入内容时，因为input中的v-model绑定了message，所以会实时将输入的内容传递给message，message发生改变。

  • 当message发生改变时，我们使用Mustache语法，将message值插入DOM，DOM发生响应改变

v-model原理

v-model其实是一个语法糖，背后本质是包含两个操作：

  • 1.v-bind绑定一个value属性
  • 2.v-on指令给当前元素绑定input事件

v-model结合radio

v-model绑定radio可以省略input的name属性

v-model结合checkbox

v-model结合select

vue值绑定

动态给value赋值，就是v-bind

vue修饰符

lazy修饰符：

  • 默认情况下，v-model默认是在input事件中同步输入框的数据的。
  • 也就是说，一旦数据发生改变data中的数据就会自动发生改变
  • lazy修饰符可以让数据在失去焦点或者回车时才会更新

number修饰符：

  • 默认情况下，在输入框中无论我们输入的是字母还是数字，都会被当做字符串类型进行处理。
  • 但是如果我们希望处理的是数字类型，那么最好直接将内容当做数字处理。
  • number修饰符可以让在输入框中输入的内容自动转成数字类型

trim修饰符：

  • 如果输入的内容首尾有很多空格，通常我们希望将其去除
  • trim修饰符可以过滤内容左右两边的空格
